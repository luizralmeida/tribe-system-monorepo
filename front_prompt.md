## Contexto
Tendo a api pronta em @api, é necessário utilizar o diretório @front para iniciar um projeto em VueJs para implementar a interface do sistema de gestão de convites.

É importante que o frontend seja responsivo e desenhado pensado em práticas de mobile-first, tendo em vista que os usuários utilizarão a aplicação principalmente via mobile.

Considerando @api, é possível que um usuário tenha vínculo com N eventos. Portanto, a primeira tela visualizada pelo usuário devem ser os eventos disponíveis a ele. Caso o usuário seja super, deve visualizar todos, caso não tenha essa role deve visualizar apenas os eventos vinculados a ele.

Ao selecionar o evento que deseja, o usuário deve ser direcionado para um dashboard contendo: quantidade total de convidados e quantidade de convidados confirmados. Logo abaixo, o usuário deve conseguir visualizar uma lista paginada de seus convidados, cada item na lista deve conter nome, número de telefone e quantidade de dependentes se aplicável. Nessa listagem devem haver atalhos para criar, editar, excluir ou visualizar convidados e seus dependentes.

É importante que super usuário tenha um menu em que possa navegar entre funcionalidades, como "Eventos" e essa tela deve ser a de seleção do evento (com atalho para criação), "Usuários" onde o super usuário visualiza os usuários do sistema com seus eventos vinculados.

## Entregáveis esperados
1. Uma aplicação Vue com interface moderna e elegante que utilize das últimas versões das libs necessárias, além das melhores práticas de VueJs e TypeScript.
2. Telas que suportem o fluxo detalhado no contexto.
